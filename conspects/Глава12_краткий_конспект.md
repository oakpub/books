# Глава 12 — Краткий конспект: Обобщения (Generics)

> **Для кого**: для тех, кто умеет программировать, но не знает C# или .NET.

---

## Что такое обобщения?

**Обобщения (Generics)** — механизм многократного использования **алгоритмов** без привязки к конкретным типам данных. Разработчик пишет алгоритм один раз, а пользователь указывает нужный тип при его использовании.

```csharp
List<DateTime> dtList = new List<DateTime>();  // T = DateTime
List<String>   names  = new List<String>();    // T = String
```

**Параметр типа** (`T`) — переменная-заглушка для конкретного типа.  
**Аргумент типа** — конкретный тип, подставляемый при использовании (`DateTime`, `String`...).

---

## Зачем нужны обобщения? (4 причины)

| Преимущество | Суть |
|---|---|
| **Безопасность типов** | Компилятор не позволит добавить `String` в `List<DateTime>` |
| **Защита исходного кода** | Не нужно видеть реализацию алгоритма, чтобы его применять |
| **Чище код** | Не нужны явные приведения типов (`(DateTime)list[0]` → просто `list[0]`) |
| **Производительность** | Нет упаковки/распаковки для значимых типов; `List<int>` в 7× быстрее `ArrayList` |

---

## Обобщения в FCL

Основные пространства имён:

| Пространство имён | Содержимое |
|---|---|
| `System.Collections.Generic` | `List<T>`, `Dictionary<TKey,TValue>`, `Queue<T>`, `Stack<T>`, `HashSet<T>`... |
| `System.Collections.ObjectModel` | `ReadOnlyCollection<T>`, `ObservableCollection<T>`... |
| `System.Collections.Concurrent` | Потокобезопасные коллекции |

> Рихтер рекомендует **всегда** использовать обобщённые коллекции вместо необобщённых (`ArrayList`, `Hashtable`).

---

## Открытые и закрытые типы

- **Открытый тип** — параметры не заданы: `List<>`, `Dictionary<,>`. Нельзя создать экземпляр.
- **Закрытый тип** — все параметры заданы: `List<Int32>`, `List<String>`. Можно создать экземпляр.

Каждый закрытый тип — **отдельный тип** со своими статическими полями.

---

## Разрастание кода (Code Explosion)

CLR генерирует машинный код для каждой пары «метод + тип данных». Оптимизации:
- Один и тот же закрытый тип компилируется **только один раз** (даже в разных сборках).
- Все **ссылочные типы** используют **один и тот же** машинный код (потому что указатель = указатель).
- Для **значимых типов** — отдельный код для каждого (`Int32`, `Double`, `DateTime` — разные).

---

## Обобщённые методы

Методы могут иметь **собственные** параметры типа (независимо от типа-контейнера):

```csharp
private static void Swap<T>(ref T o1, ref T o2) {
    T temp = o1; o1 = o2; o2 = temp;
}
```

Компилятор умеет **автоматически выводить тип** (type inference):

```csharp
int a = 1, b = 2;
Swap(ref a, ref b);  // компилятор сам понимает, что T = int
```

## Вариантность: `in` и `out`

Применяется к параметрам типа **делегатов и интерфейсов**:

| Ключевое слово | Термин | Смысл |
|---|---|---|
| `out T` | Ковариантность | `T` — только на выходе (возвращаемое значение). Можно приводить `производный → базовый`. |
| `in T` | Контравариантность | `T` — только на входе (параметры). Можно приводить `базовый → производный`. |

> Вариантность работает **только для ссылочных типов** — значимые (int, DateTime) не поддерживаются.

## Ограничения (Constraints)

Ограничения указывают, какие типы допустимы в качестве аргумента типа. Это расширяет возможности работы с `T`:

```csharp
public static T Min<T>(T o1, T o2) where T : IComparable<T> {
    return o1.CompareTo(o2) < 0 ? o1 : o2;
}
```

### Виды ограничений:

| Синтаксис | Тип | Смысл |
|---|---|---|
| `where T : class` | Основное | T — любой ссылочный тип |
| `where T : struct` | Основное | T — любой значимый тип (кроме Nullable) |
| `where T : SomeClass` | Основное | T — SomeClass или производный от него |
| `where T : IFoo` | Дополнительное | T должен реализовывать интерфейс IFoo |
| `where T : new()` | Конструктора | T должен иметь публичный конструктор без параметров |
| `where T : TOther` | Дополнительное | T должен быть совместим с TOther |

### Ключевое слово `default`

```csharp
T temp = default(T); // null для ссылочных, 0/false для значимых
```

---

## Запомнить

- Обобщения существуют **на уровне CLR**, а не только как синтаксический сахар.
- Нельзя перегружать типы/методы по ограничениям — только по **арности** (числу параметров типа).
- При переопределении обобщённого виртуального метода ограничения **наследуются**, задавать новые нельзя.
