# Глава 12 — Вопросы для самопроверки: Обобщения (Generics)

---

## Базовый уровень

**1.** Что такое обобщения (generics)? Чем они отличаются от простого наследования как инструмента повторного использования кода?

**2.** Что такое «параметр типа» (type parameter) и что такое «аргумент типа» (type argument)? Приведите пример.

**3.** Назовите четыре главных преимущества обобщений перед необобщёнными алгоритмами (например, перед `ArrayList`).

**4.** В каких пространствах имён FCL находятся основные обобщённые коллекции? Перечислите хотя бы три конкретных класса.

**5.** Почему Рихтер рекомендует отказаться от необобщённых классов (`ArrayList`, `Hashtable`) в пользу обобщённых?

**6.** Что такое «открытый тип» (open type) и «закрытый тип» (closed type)? Можно ли создать экземпляр открытого типа?

**7.** Что выведет компилятор, если задать такое ограничение: `where T : struct, new()`? Почему?

## Средний уровень

**8.** Дан код:
```csharp
ArrayList a = new ArrayList();
for (int i = 0; i < 1_000_000; i++) a.Add(i);
```
```csharp
List<int> l = new List<int>();
for (int i = 0; i < 1_000_000; i++) l.Add(i);
```
Почему второй вариант быстрее и создаёт меньше нагрузки на сборщик мусора? Объясните механизм.

**9.** Чем отличаются статические поля `List<Int32>` и `List<String>`? Разделяют ли они одно поле или у каждого типа своё?

**10.** Почему для **ссылочных** типов `List<String>` и `List<Stream>` CLR использует **один** машинный код, а для значимых типов `List<Int32>` и `List<DateTime>` — **разный**?

**11.** Что такое «разрастание кода» (code explosion)? Как CLR борется с ним?

**12.** Можно ли создать `List<DateTime>` и `List<String>` и присвоить второй из них переменной типа `List<Object>`? Почему?

```csharp
List<Object> lo = new List<String>(); // скомпилируется?
```

**13.** Что делает `default(T)` и зачем оно нужно? Что вернёт `default(int)` и `default(string)`?

**14.** Компилируется ли следующий код? Если нет — почему? Как исправить?
```csharp
private static T Min<T>(T o1, T o2) {
    if (o1.CompareTo(o2) < 0) return o1;
    return o2;
}
```

**15.** Объясните разницу между основным ограничением (`where T : class`), дополнительным ограничением (`where T : IComparable<T>`) и ограничением конструктора (`where T : new()`). Сколько каждого вида можно задать?

## Продвинутый уровень

**16.** Что означают ключевые слова `in` и `out` в параметрах обобщённых делегатов и интерфейсов? Приведите пример использования.

**17.** Объясните, почему следующий код работает, а следующий за ним — нет:
```csharp
// Работает:
IEnumerable<Object> seq = new List<String>();

// Не работает:
IEnumerable<Object> seq = new List<DateTime>(); // ошибка компиляции
```

**18.** Почему нельзя перегружать методы по ограничениям? Что из следующего является ошибкой?
```csharp
internal sealed class AType<T> { }
internal sealed class AType<T> where T : IComparable<T> { } // ошибка?
```

**19.** При переопределении обобщённого виртуального метода можно ли задать **новые** ограничения для параметров типа? Что происходит на практике?
```csharp
internal class Base {
    public virtual void M<T1>() where T1 : struct { }
}
internal sealed class Derived : Base {
    public override void M<T3>() where T3 : EventArgs { } // скомпилируется?
}
```

**20.** Компилятор C# отдаёт приоритет необобщённым методам перед обобщёнными при выборе перегрузки. Покажите сценарий, где это важно — в частности, почему неправильная привязка могла бы вызвать бесконечную рекурсию.

**21.** Что такое выведение типов (type inference) при вызове обобщённых методов? Почему следующий вызов приведёт к ошибке компиляции?
```csharp
string s1 = "A";
object s2 = "B";
Swap(ref s1, ref s2); // ошибка?
```

**22.** Может ли перечислимый тип (`enum`) быть аргументом типа при ограничении `where T : struct`? Почему CLR не позволяет ограничить тип только перечислимыми типами напрямую через constraints?

**23.** Почему `Nullable<Nullable<int>>` является недопустимым типом в CLR? Связано ли это с ограничением `struct` у `Nullable<T>`?

## Ответы — подсказки

> Используйте эти подсказки, если не можете вспомнить ответ.

| Вопрос | Ключевые слова для поиска |
|---|---|
| 1 | алгоритмы, наследование = поведение, обобщения = типонезависимые алгоритмы |
| 3 | тип безопасность, защита кода, нет касты, нет упаковки |
| 7 | все struct имеют конструктор без параметров — `new()` избыточно |
| 8 | boxing/unboxing, GC pressure, значимые типы в куче |
| 9 | объект-тип (type object), статические поля, закрытый тип |
| 10 | указатель = указатель для ссылочных; размер и инструкции разные для значимых |
| 11 | JIT per type+method, оптимизация: один код для всех ref-типов |
| 12 | инвариантность List<T>; IEnumerable<T> — ковариантен через `out T` |
| 16 | ковариантность = `out`, контравариантность = `in`, только ref-типы |
| 19 | ограничения наследуются, задавать новые нельзя — ошибка CS0460 |
| 23 | Nullable<T> where T : struct — Nullable<Nullable<T>> рекурсивен |
