# Глава 6 — Краткий конспект: Члены и типы в C# / CLR

> **Для кого**: для тех, кто умеет программировать, но не знает C#.

---

## Что такое «тип» и «члены»?

В C#/CLR **тип** — это то же самое, что класс (или структура) в любом ООП-языке. Внутри типа живут **члены** — его составные части:

| Член | Аналог в других языках | Суть |
|---|---|---|
| **Константа** | `const`, `UPPER_CASE` | Именованное неизменяемое значение, всегда «прицеплено» к типу (статическая) |
| **Поле** | переменная экземпляра / статическая переменная | Хранит данные объекта или класса |
| **Конструктор экземпляра** | `__init__` / конструктор | Инициализирует поля при создании объекта |
| **Конструктор типа** | статический инициализатор | Инициализирует статические поля один раз при первом использовании типа |
| **Метод** | функция / метод | Выполняет действия над объектом или типом |
| **Перегруженный оператор** | `__add__`, `operator+` | Задаёт поведение `+`, `-`, `==` и т.д. для объектов |
| **Оператор преобразования** | явное/неявное приведение | Превращает объект одного типа в другой |
| **Свойство** | геттер/сеттер | Выглядит как поле, но внутри — методы (можно добавить проверку) |
| **Событие** | listener / callback-регистратор | Позволяет подписаться на «уведомление» от объекта |
| **Вложенный тип** | inner class | Тип внутри другого типа |

---

## Видимость типа

Два уровня для типов верхнего уровня:
- **`public`** — доступен из любой сборки (библиотеки/exe)
- **`internal`** (по умолчанию в C#) — доступен только внутри своей сборки

---

## Дружественные сборки (`InternalsVisibleTo`)

Проблема: хочу дать доступ к `internal`-типам одной конкретной сборке (например, для unit-тестов), но не всем подряд.

Решение: атрибут `[assembly: InternalsVisibleTo("НазваниеСборки")]` — открывает внутренние типы только указанной сборке.

---

## Модификаторы доступа к членам (от строгого к открытому)

| C# | Кто видит |
|---|---|
| `private` | только сам класс |
| `protected` | класс + наследники (в любой сборке) |
| `internal` | любой код в той же сборке |
| `protected internal` | наследники ИЛИ та же сборка |
| `public` | все и везде |

По умолчанию C# выбирает **`private`** — самый строгий.

---

## Статические классы

Класс с ключевым словом `static` — это «неинстанцируемая утилита» (как `Math`, `Console`):
- нельзя создать объект (`new`)
- только статические члены
- нельзя наследоваться от него

---

## Частичные типы (`partial`)

Ключевое слово `partial` позволяет разбить код одного класса на **несколько файлов**. Компилятор их объединяет. Удобно:
- при командной разработке
- когда часть кода генерируется автоматически (Visual Studio, генераторы)

---

## Виртуальные методы и полиморфизм

- **Невиртуальный метод** — вызывается точно по типу переменной, быстрее.
- **Виртуальный метод** (`virtual`) — вызывается по реальному типу объекта, медленнее (поиск через таблицу методов).
- **`override`** — переопределить виртуальный метод в наследнике.
- **`sealed`** на метод — запретить дальнейшее переопределение.
- **`new`** — объявить метод в наследнике, который НЕ связан с одноимённым методом базового класса (скрытие, а не полиморфизм).
- **`abstract`** — метод без реализации, обязателен к переопределению.

---

## Управление версиями компонентов

.NET-сборка — это версионированный компонент (`1.2.3.4` = major.minor.build.revision). Служебные версии (патчи) меняют только build/revision; новые несовместимые версии меняют major/minor.

---

## Практические советы от Рихтера

- **Поля — всегда `private`**, доступ через свойства.
- **Методы — по умолчанию `private` и невиртуальные**, открывать только нужное.
- **Классы — лучше `sealed`** (запечатанные): безопаснее, быстрее, проще версионировать.
- Виртуальных методов должно быть как можно **меньше** — они медленнее и усложняют совместимость.
